Q-2:Difference between Local Storage, Session Storage, and Cookies?

Local storage: The stored data is not deleted when the browser is closed.


Session Storage: The data is stored only for the session and will be deleted when the browser is closed.


Cookies: The data can be set to expire at a certain time.

Q-3:What is event loop

The Event Loop has one simple job — to monitor the Call Stack and the Callback Queue. If the Call Stack is empty, the Event Loop will take the first event from the queue and will push it to the Call Stack, which effectively runs it. Such an iteration is called a tick in the Event Loop.

Analyzing the code and detailing how it works

1: quizTimer function

The quick timer function takes a parameter and checks if it is true. If it is true, it clears the interval and returns. If the parameter is false, it sets an interval and captures an HTML element with the ID "count". It calculates the minutes and seconds from the count variable and formats them with leading zeroes if necessary. It then sets the innerHTML of the countHtml element to the formatted time with a subscript "sec" element. It increments the count variable and changes the text color every second. The interval runs every 1 second.
